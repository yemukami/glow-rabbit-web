# 懺悔と再発防止チェックリスト

作業開始前に必読。ミスが発覚したら以下に従い、`REMORSE_LOG.md` へ具体的な懺悔と再発防止策を追記する。

1. **事象を一文で明記**: 何が起きたか、誰が困るかを簡潔に。
2. **原因の特定**: 直接原因と背景（判断/プロセス/テスト欠如）を分けて書く。
3. **再発防止策**: 具体的な手順・ガード・テスト追加など、行動ベースで列挙。
4. **テスト/確認**: 再発防止のために何を確認したか（テスト名や手動手順）。
5. **進行中の注意**: 同種変更に入る前に本ファイルと `REMORSE_LOG.md` を読むこと。

テンプレ（コピーして `REMORSE_LOG.md` に追記）
```
- 事象: ...
- 原因: ...
- 再発防止: ...
- 確認: ...
```

LLM運用上の戒め:
- START/同期/stopRunner/ペース送信を動かす前に必ず SYNC_START_SPEC.md と REMORSE_LOG.md を再読。
- あいまい・「可能性」ベースの変更は必ず人間に確認し、独断で仕様を揺らさない。
